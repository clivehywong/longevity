# Archive

This directory holds historical material that may still be useful for context but is not the primary maintained documentation.

## What belongs here

- old run-status notes
- implementation summaries
- archived plans
- job logs
- one-off incident writeups

## How to use it

Start in `docs/user/` or `docs/developer/` first. Only come here when you need past context that has not yet been generalized into the maintained docs.
